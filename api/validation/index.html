    Advanced Cross-Validation¶

    scomp_link.validation.advanced_cv.AdvancedCV ¶

    AdvancedCV provides advanced cross-validation strategies beyond standard K-Fold.

    Dependencies: - numpy - sklearn.model_selection - sklearn.utils

    Methods: - loocv: Leave-One-Out Cross Validation - bootstrap: Bootstrap resampling validation - evaluate_all: Run all validation strategies

    loocv(model, X, y, scoring=None) staticmethod ¶

    Leave-One-Out Cross Validation

    Parameters: - model: Estimator to validate - X: Features - y: Target - scoring: Scoring metric (default: accuracy for classification, r2 for regression)

    Returns: - dict with mean, std, and all scores

    bootstrap(model, X, y, n_iterations=1000, test_size=0.3, random_state=42) staticmethod ¶

    Bootstrap validation with out-of-bag evaluation

    Parameters: - model: Estimator to validate - X: Features - y: Target - n_iterations: Number of bootstrap samples - test_size: Proportion of data for testing - random_state: Random seed

    Returns: - dict with mean, std, and all scores

    evaluate_all(model, X, y, include_loocv=True, include_bootstrap=True, bootstrap_iterations=1000, scoring=None) staticmethod ¶

    Run all advanced validation strategies

    Parameters: - model: Estimator to validate - X: Features - y: Target - include_loocv: Whether to run LOOCV (can be slow for large datasets) - include_bootstrap: Whether to run Bootstrap - bootstrap_iterations: Number of bootstrap samples - scoring: Scoring metric

    Returns: - dict with results from all methods

    Fairness Metrics¶

    scomp_link.validation.fairness.FairnessMetrics ¶

    Compute fairness and bias metrics for classification models.

    Dependencies: numpy, pandas, plotly

    Parameters:

    Name Type Description Default
    1. y_true

    ground truth labels (binary 0/1)

    		 required
    2. y_pred

    predicted labels (binary 0/1)

    		 required
    3. sensitive_feature

    array/series of protected attribute values

    		 required
    4. positive_label

    which label is considered "positive" (default 1)

    		 required
    Usage example

    fm = FairnessMetrics(y_true, y_pred, sensitive_feature=df['gender']) report = fm.compute_all() fig = fm.plot_fairness_report(report)

    demographic_parity() ¶

    Demographic Parity: P(Y_hat=1 | A=a) should be equal across groups. Returns selection rate per group + ratio (min/max).

    equalized_odds() ¶

    Equalized Odds: P(Y_hat=1 | Y=y, A=a) should be equal across groups for both y=0 and y=1. Returns TPR and FPR per group.

    disparate_impact() ¶

    Disparate Impact: ratio of selection rates between unprivileged and privileged groups. The 4/5 rule: ratio should be >= 0.8.

    equal_opportunity() ¶

    Equal Opportunity: P(Y_hat=1 | Y=1, A=a) should be equal across groups. (Only looks at TPR, not FPR.)

    compute_all() ¶

    Compute all fairness metrics at once.

    summary(report=None) ¶

    Summary table of fairness metrics.

    plot_fairness_report(report=None) ¶

    Plotly grouped bar chart comparing metrics across groups.
